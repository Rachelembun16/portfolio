Use Case Diagram

Use Case Diagrams are used to explain the functions that can be performed by users on this system. The diagram shows the users in the system and also the distribution of access rights for each user.

Use Case Diagrams are built after doing an analysis such as needs analysis, pieces analysis, weakness analysis, also competitor analysis. Once the analytical design is approved, a use case diagram is generated to visualize the idea, so that it can be understood by other team members.

Activity Diagram

Activity diagrams are used to describe the flow of several functions in this system. Every function defined in the use case diagram, must be developed into an activity diagram.

Activity diagrams are used to help developers understand concepts and how a system works. The diagram beside is one of the activity diagrams built in this project.

ER Diagram

ER Diagram is a diagram that shows the database design used in the system. ER diagram shows the relationship of one entity with other entities.

An ER diagram contains database fields and rows, so we can build a database just by looking at the ER diagram. ER Diagrams are created to guide back end engineers in developing databases.
